Healthcare Appointment No-Show API

This Django REST Framework application analyses healthcare appointment attendance, patient medical conditions, waiting time and no-show behaviour using the Healthcare Appointment No-Show dataset.

The API uses HyperlinkedModelSerializer and a small number of reusable RESTful resource endpoints. Clients can combine query parameters such as age range, medical conditions, attendance status and waiting time instead of relying on multiple hardcoded endpoints.

Patient API and Example Queries

All patient queries use the same /api/patients/ endpoint. The examples below demonstrate how callers can choose their own filters.

GET / POST
Patient Collection

Retrieve patient records or create a new patient using POST.

GET / PUT / PATCH / DELETE
Patient Detail

Retrieve, update or delete one patient record.

FILTER EXAMPLE
Patients by Age Range

The caller chooses the minimum and maximum ages.
?min_age=40&max_age=60

FILTER EXAMPLE
Patients by Medical Condition

Filter using Boolean health fields.
?diabetes=true
?hypertension=true

COMBINED FILTERS
Combined Patient Filters

Combine age and medical-condition filters.
?min_age=65
&diabetes=true
&hypertension=true

RESULT CONTROL
Limit Patient Results

The caller chooses how many records to return.
?limit=20

Appointment API and Example Queries

All appointment searches use the same /api/appointments/ endpoint. Filters can be combined to answer different healthcare questions.

GET / POST
Appointment Collection

Retrieve appointment records or create a new appointment.

GET / PUT / PATCH / DELETE
Appointment Detail

Retrieve, update or delete one appointment record.

FILTER EXAMPLE
Attendance Status

Return appointments where patients did not attend.
?showed_up=false

COMBINED FILTERS
SMS and No-Show Analysis

Find missed appointments where an SMS reminder was received.
?showed_up=false
&sms_received=true

COMBINED FILTERS
Patient Risk and Attendance

Combine attendance, age and medical-condition filters.
?showed_up=false
&min_age=65
&diabetes=true

FILTER EXAMPLE
Waiting-Time Range

The caller chooses the minimum and maximum waiting time.
?min_wait_days=10
&max_wait_days=30

FILTER EXAMPLE
Appointments by Neighbourhood

Search appointment records using part of a neighbourhood name.
?neighbourhood=JARDIM

RESULT CONTROL
Limit Appointment Results

Return a caller-selected number of appointments.
?limit=20
